## Design Context

### Users
WinMux is for a power user working in dense, multi-pane developer sessions on Windows. The product is used for active project work: terminals, browser previews, diffs, editor panes, thread switching, and review inside one compact workspace. The primary job is staying in flow while moving between code, context, and shell operations without decorative chrome getting in the way.

### Brand Personality
Calm, precise, utilitarian. The interface should feel intentional, composed, and trustworthy rather than playful or glossy. It should read like a serious tool for focused work, with subtle confidence rather than marketing energy.

### Aesthetic Direction
Subtractive desktop tool design. Flat surfaces, cooler neutrals, restrained accent color, and diff-inspired metadata language. Avoid glossy pills, nested cards, bloated padding, stock WinUI heaviness, and ornamental containers. Favor direct rows, separators, layered states, and information density. The app should feel closer to a refined editor/workspace shell than a generic settings dashboard.

### Design Principles
1. Remove containers that do not earn their space.
2. Prefer rows, dividers, and alignment over cards inside cards.
3. Keep the shell calm: low-gloss, low-chrome, high-clarity.
4. Use color sparingly and semantically, especially for file/review/diff state.
5. Fit more meaning into the same window size without sacrificing readability.
